# 开发流程规范

开发过程中，经常有各种各样的问题，

- a把b的代码冲掉了
- a打的测试包被上线了
- a写的功能没有完整的测试用例

等等等等，如果没有一定的协作规范，那么我们的工作就是低效的、低质量的。

所以我们需要一定的流程规范来约束我们的行为。规范涉及面包含：代码规范、提交规范、合并规范、测试打包规范等等等等

同时，我们要了解软件研发的整个生命周期，了解每一步都该干什么，以保证质量
